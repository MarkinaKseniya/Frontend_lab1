Задание 1.3. Найти в интернете описание работы DNS, узнать, что такое: ns-сервер, делегирование домена, A-запись, MX-запись.

DNS (Domain Name System) — это система, которая преобразует доменные имена в соответствующие IP-адреса. Она является неотъемлемой частью работы Интернета.

Работа DNS происходит по следующему принципу: когда вы вводите доменное имя в браузере, ваш компьютер отправляет запрос на DNS-сервер, который содержит базу данных доменных имен и их соответствующие IP-адреса. DNS-сервер выполняет поиск в этой базе данных и, если находит соответствие, возвращает IP-адрес обратно на компьютер пользователя. Затем браузер использует этот IP-адрес для связи с сервером, где хранится запрашиваемый веб-сайт, и отображает его пользователю.

Важно отметить, что DNS — это распределенная система сетевых серверов по всему миру. Когда ваш запрос попадает на DNS-сервер, который не содержит информацию о запрашиваемом доменном имени, он перенаправляет запрос на другой DNS-сервер, пока не будет найдено соответствие. Это позволяет обеспечить быстрый и точный перевод доменных имен в IP-адреса.

NS-сервер (Name Server) — это сервер, который ответственен за хранение информации о доменном имени и его соответствующем IP-адресе. NS-серверы участвуют в процессе разрешения доменных имен. Когда компьютер отправляет запрос на DNS-сервер для разрешения доменного имени, NS-серверы возвращают информацию о запрашиваемом домене.

Делегирование домена — это процесс передачи управления доменом от одного DNS-сервера к другому. Когда вы регистрируете новый домен, вы назначаете NS-сервера, которые будут отвечать за разрешение этого домена. Таким образом, делегирование домена позволяет указать, какие NS-серверы будут служить источником информации о вашем домене.

A-запись (Address Record) — это тип DNS-записи, которая устанавливает соответствие между доменным именем и его IP-адресом. A-запись используется для перевода доменного имени в конкретный IP-адрес, необходимый для связи с сервером, где хранится веб-сайт.

MX-запись (Mail Exchanger Record) — это тип DNS-записи, который определяет, какие серверы должны принимать электронные письма, адресованные домену. MX-запись указывает на адреса электронной почты, которые отвечают за обработку и доставку сообщений почты для данного домена.

Использование MX-записей позволяет эффективно маршрутизировать почту, когда у домена есть несколько почтовых серверов, обрабатывающих входящую почту. DNS-сервер сверяет MX-записи и направляет почтовые серверы к правильному серверу для доставки сообщений.
